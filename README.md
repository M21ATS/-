# توثيق مشروع لعبة "حروف مع حمودي" (Hroof with Hammoudi)

هذا المستند يقدم شرحاً تفصيلياً وشاملاً لمشروع لعبة "حروف مع حمودي"، وهو مصمم ليكون مرجعاً تقنياً للمطورين لفهم بنية المشروع، منطق اللعبة، وكيفية تشغيله وتطويره.

---

## 1. نظرة عامة على المشروع (Project Overview)

**الفكرة الأساسية:**
اللعبة مستوحاة من البرنامج الشهير "حروف"، وهي لعبة تعليمية تفاعلية تعتمد على الثقافة العامة والسرعة. يتنافس اللاعبون (أو الفرق) على لوحة مكونة من 25 حرفاً موزعة في شبكة (5x5).

**آلية العمل:**
- يختار اللاعب حرفاً من اللوحة.
- يظهر سؤال تبدأ إجابته بهذا الحرف.
- في حال الإجابة الصحيحة، يتلون الحرف بلون اللاعب (أزرق أو أحمر).
- الهدف هو تكوين خط متصل من الحروف الملونة (أفقياً، عمودياً، أو قطرياً) للفوز.
- تدعم اللعبة اللعب الفردي (ضد الوقت) واللعب الجماعي (عبر الغرف).

---

## 2. واجهة المستخدم والتصميم (UI/UX & Design)

**نمط الـ Rubber Hose:**
تم تطبيق النمط الفني المعروف بـ "Rubber Hose" (الذي يتميز بالأطراف المرنة والخطوط السميكة والمنحنيات الناعمة) من خلال:
- **الحواف الدائرية الكبيرة (Rounded-2xl/3xl):** تعطي شعوراً بالليونة والودية.
- **الحدود السميكة (Thick Borders):** استخدام حدود واضحة (Border-4) لتمييز العناصر.
- **الألوان الحيوية:** استخدام لوحة ألوان مشبعة (Indigo, Rose, Amber) لتعزيز الطابع الكرتوني.

**التحريكات (Animations):**
تم استخدام مكتبة `motion` (Framer Motion) لتحقيق تفاعل سلس:
- **تأثيرات الارتداد (Spring Physics):** عند فتح النوافذ المنبثقة أو الضغط على الحروف، يتم استخدام `type: "spring"` ليعطي حركة مرنة تشبه المطاط.
- **الانتقالات (Transitions):** استخدام `AnimatePresence` لضمان دخول وخروج العناصر بشكل ناعم.
- **ردود الفعل البصرية:** الحروف تهتز أو تكبر قليلاً عند التفاعل معها لتعزيز تجربة المستخدم.

---

## 3. المنطق البرمجي (Game Logic)

المشروع مبني باستخدام **React** و **TypeScript**، وينقسم المنطق إلى:

### أ. إدارة الأسئلة (Question Management):
- يتم تخزين الأسئلة في كائنات ضخمة (`PRESET_BANKS`) مقسمة حسب الفئة (حيوانات، حكم، عواصم، إلخ) ثم حسب الحرف.
- عند اختيار حرف، يتم جلب سؤال عشوائي من البنك المخصص لذلك الحرف.

### ب. التحقق من الإجابات:
- يتم تنظيف الإجابة المدخلة من المسافات والرموز الزائدة.
- تتم مقارنة الإجابة مع النص المخزن في قاعدة البيانات.

### ج. نظام الغرف (Multiplayer Logic):
- يعتمد على **Socket.io** للمزامنة اللحظية.
- **الخادم (Server):** يدير الغرف، قائمة اللاعبين، وحالة اللوحة (Tiles State).
- **العميل (Client):** يرسل التحديثات (مثل تلوين حرف) ويستقبل الحالة الجديدة ليعرضها لجميع اللاعبين في نفس الغرفة.

---

## 4. الخوادم والاستضافة (Server & Hosting)

**بيئة الخادم (Server Environment):**
- **Node.js & Express:** لإدارة طلبات الـ API وخدمة الملفات الثابتة.
- **SQLite (Better-SQLite3):** قاعدة بيانات خفيفة لتخزين حسابات المستخدمين وبنوك الأسئلة المخصصة.
- **Socket.io:** للاتصال ثنائي الاتجاه بين اللاعبين.

**التجهيز للاستضافة (Render / Railway):**
- **المنافذ (Ports):** السيرفر مبرمج ليأخذ المنفذ من متغير البيئة `process.env.PORT` أو يستخدم 3000 افتراضياً.
- **قاعدة البيانات:** تم إعداد مسار ديناميكي لقاعدة البيانات (`DATABASE_PATH`) لضمان استمرار البيانات في بيئات الاستضافة التي تدعم الأقراص الثابتة (Persistent Disks) مثل `/data/hroof.db`.
- **الملفات الثابتة:** في وضع الإنتاج (`NODE_ENV=production`)، يقوم السيرفر بخدمة مجلد `dist` الذي يحتوي على ملفات React المبنية.

---

## 5. دليل التشغيل المحلي (Local Setup Guide)

لتشغيل المشروع على جهازك الخاص للتطوير، اتبع الخطوات التالية:

1. **المتطلبات الأساسية:**
   - تثبيت **Node.js** (إصدار 18 أو أحدث).
   - محرر أكواد (مثل VS Code).

2. **تثبيت المكتبات:**
   افتح الطرفية (Terminal) في مجلد المشروع ونفذ:
   ```bash
   npm install
   ```

3. **تشغيل وضع التطوير:**
   نفذ الأمر التالي لتشغيل السيرفر وواجهة المستخدم معاً:
   ```bash
   npm run dev
   ```
   سيظهر لك رابط (غالباً `http://localhost:3000`)، قم بفتحه في المتصفح.

4. **بناء المشروع للإنتاج:**
   إذا أردت تجربة النسخة النهائية:
   ```bash
   npm run build
   npm start
   ```
